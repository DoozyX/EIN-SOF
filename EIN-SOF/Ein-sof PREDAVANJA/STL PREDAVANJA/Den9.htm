4 Справување со исклучоци

 

Исклучок (exception) означува проблем што настанал при извршување на програма. Терминот исклучок е одбран поради фактот што проблем при извршување е настан што се случува ретко. Exception handling претставува начин за справување со исклучоците, односно им овозможува на програмите да продолжат да се извршуваат иако се појавила грешка.

Кај традиционалното програмирање често има мешање на кодот за проверка на грешки со кодот за решавање на програмската задача, како во следниот псевдокод:

 

Изврши задача

Ако претходната задача не се извршила коректно

               Процесирај ја грешката

Изврши задача

Ако претходната задача не се извршила коректно

               Процесирај ја грешката

               . . .

Ваквиот начин за справување со грешка ја прави програмата тешка за следење, менување и одржување, особено кај големи апликации.

Exception handling овозможува отстранување на кодот за справување со грешки од текот на програмата. Програмерот може да одбере кои грешки ќе ги земе предвид. Се зголемува отпорноста на грешки (fault tolerance) на програмата.

Exception handling е дизајниран за процесирање на синхрони грешки, како што се: надминување на граници на поле, надминување на капацитет на променлива, делење со нула, погрешни функциски параметри, недостаток на меморија итн. Асинхрони настани, како В/И операции, кликање на глушец и сл., што се случуваат паралелно со програмскиот тек, не се погодни за обработка со exception handling. 

Важно е стратегијата за справување со исклучоци да се определи однапред и да се вгради уште на самиот почеток, а не откако ќе се имплeментира апликацијата.

Овој механизам може да се употреби за справување со грешки при повикување на функции членки, конструктори и класи.

За разлика од класичната проверка на грешки, ако грешките не се случуваат често, exception handling не внесува додатен трошок при извршување ако не настануваат грешки. Но, ако некоја грешка се појавува барем 30% од времето, таа треба да се обработи во самата програма.

Традиционални техники за справување со грешки

Пишување на робусен код кој е отпорен на грешки е напорна работа. Обработување на сите грешки значително го зголемува кодот и го прави многу потешко читлив. Затоа најчесто грешките се игнорираат. Но, игнорирање на грешка може да предизвика непредвидливо однесување на програмата, дури и прекин на извршување, што е неприфатливо кај комерцијални продукти.

Во C, може да се применат следните техники:

1.      Информацијата за грешката се враќа на повикувачката функција или се поставува глобално знаменце. Сите стандардни функции го прават тоа, но таа информација ретко се користи. Исто така, продолжување на извршување на функција во која  настанал исклучок нема смисол.

2.      Употреба на функции од стандардна библиотека за сигнализација (signal и raise). Овој систем е сложен и малку користен, бара тесна поврзаност помеѓу точката каде настанала грешката и местото каде што се обработува.

3.      Со функцијата setjmp() се зачувува позната “добра“ состојба на програмата, а потоа со longjmp() таа се обновува и се врши скок кај сегментот што ја обработува грешката. Овие функции не повикуваат деструктори, па ако скокот се изврши во опсег каде што треба да се уништи објект, однесувањето на програмата не може да се предвиди, бидејќи може да останат алоцирани објекти чија важност престанала. Практично е невозможно да се врати претходната состојба на програмата.

Заеднички проблем на овие шеми е големата поврзаност помеѓу повикувачката функција и справувачот со грешки. 

Exception handling - справување со грешки во C++

Во C++ е воведена нова техника за справување со грешки која се нарекува exception handling.

  • Кодот кај exception handling не се преплетува со останатиот код. Повикувачката функција е справувачот со грешки се одвоени.
  • Грешките не можат да се игнорираат. Ако функција сака да испрати порака за грешка, таа фрла објект кој ја претставува таа грешка. Тој е потоа фатен од справувачот со грешки или прекинува програмата.

Exception handling дава начин на кој компонентите можат да известат за проблемот што настанал при извршување. Тој се употребува кај функции што откриваат грешка, но не може да се справат со неа. Тогаш функцијата фрла исклучок (throws an exception). Нема гаранција дека секогаш постои exception handler кој го фаќа (catch) исклучокот, но ако има, тој го презема справувањето со проблемот. Исклучок кој не е фатен резултира со прекин на извршувањето на програмата.

Кога се применува exception handling во C++, деловите во кои може да настане исклучок се пишуваат во try блок. Барем еден catch блок мора да следи по него. Во секој catch блок е специфицирано каков тип на исклучок тој процесира. cactch блокот е всушност exception handler.

Кога во try блокот ќе настане грешка, тогаш неговото извршување се прекинува веднаш и извршувањето се пренесува на catch блокот. Сите променливи што немаат важност надвор од доменот на овој блок се уништуваат. Се извршува првиот catch блок чии параметри одговараат на грешката. Останатите catch блокови што го имаат истиот параметар се игнорираат. Кога ќе заврши catch блокот, сите променливи што немаат важност надвор од неговиот домен се уништуваат. Потоа, извршувањето се пренесува на првата наредба по try-catch секвенцата. Ако не дојде до исклучок, се игнорираат catch блоковите и по завршувањето на try блокот извршувањето се пренесува на првата наредба по try-catch секвенцата.

 Следниот е демнстрација на exception handling. Тој користи exception handling за справување со исклучоци кои настануваат при делење со нула.

 

 

// Primer za exception-handling koj

// proveruva dali se pojavuva delenje so 0

#include <iostream>

 

using std::cout;

using std::cin;

using std::endl;

 

#include <exception>

 

using std::exception;

//funkcii koi detektiraat delenje so 0 treba da frlat

// objekt DivideByZeroException

class DivideByZeroException : public exception {

 

public:

 

   // konstructorot ja opredeluva default error message

   DivideByZeroException::DivideByZeroException()

      : exception( "obid za delenje so nula" ) {}

 

};  // end class DivideByZeroException

 

// izvrsi delenje i frli DivideByZeroException objekt

// ako nastane delenje so nula

double quotient( int numerator, int denominator )

{

   // frli DivideByZeroException

   if ( denominator == 0 )

      throw DivideByZeroException(); // prekin na funkcijata

 

  

   return static_cast< double >( numerator ) / denominator;

 

}  // kraj na funkcijata quotient

 

int main()

{

   int number1;    // delenik

   int number2;    // delitel

   double result;  // kolicnik

 

   cout << "Vnesi dva celi broja (end-of-file za kraj): ";

 

  

   while ( cin >> number1 >> number2 ) {

  

      // try block sodrzi kod sto moze da frli isklucok

     

      try {

         result = quotient( number1, number2 );

         cout << "Kolicnikot e: " << result << endl;

 

      } // kraj na try

 

      // exception handler za isklucok delenje so nula

      catch ( DivideByZeroException &divideByZeroException ) {

         cout << "Nastanal isklucok: " <<

            divideByZeroException.what() << endl;

 

      } // kraj na catch

 

      cout << "\nVnesi dva celi broja (end-of-file za kraj): ";

 

   } 

 

   cout << endl;

 

   return 0; 

 

}  // kraj na main

4

Kolicnikot e: 0.75

 

Vnesi dva celi broja (end-of-file za kraj): 5

0

Nastanal isklucok: obid za delenje so nula

 

Vnesi dva celi broja (end-of-file za kraj): 3

Класата DivideByZeroException наследува од класата exception која е дел од стандардната библиотека на C++.

За фрлање на исклучок се користи клучниот збор throw по што следи операнд, кој во случајов е објект од класата DivideByZeroException. Параметар на throw може да биде објект од било кој тип, резултат од израз, а може да се фрли и исклучок без параметар. Како резултат на throw се создава throw операнд со кој се иницијализира catch handler-от.

Во примерот throw наредбата креира објект од класата DivideByZeroException. Неговиот конструктор со exception( "obid za delenje so nula" )му предава стринг на конструкторот од основната класа exception кој е всушност пораката за грешка за овој исклучок. Кога се фрла исклучокот quotient прекинува веднаш, пред да биде извршено делењето.  Со:

      catch ( DivideByZeroException &divideByZeroException )

се врши “фаќање“ на референца на DivideByZeroException. Со тоа се избегнува додатниот трошок за создавање на нов објект.

Соодветната порака за грешка се печати со

cout << "Nastanal isklucok: " <<

            divideByZeroException.what() << endl;

која е функција членка на exception.

Повторно фрлање на исклучок

 Може да се случи exception handler да не може да се справи со исклучок целосно. Во тој случај тој може задачата да ја префрли на друг exception handler. Тоа се нарекува повторно фрлање на исклучок (rethrowing an exception) и се постига со наредбата:

throw;

 Овој исклучок го детектира следниот try блок. Со исклучокот се справува следниот catch блок кој е на поглобално ниво.

 Следниот пример содржи повторно фрлање на исклучок:

 

// Demonstrira exception rethrowing.

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <exception>

 

using std::exception;

 

// throw, catch i rethrow

void throwException()

{

  

   try {

      cout << "  Funkcijata throwException frla isklucok\n";

      throw exception(); 

 

   }

 

 

   catch ( exception &caughtException ) {

      cout << "  Isklucok obraboten vo funkcijata throwException"

           << "\n  Funkcijata throwException povtorno go frla isklucokot";

 

      throw; 

 

   }

 

   cout << "Ova ne treba da se otpecati\n";

 

}  // kraj na throwException

 

int main()

{

  

   try {

      cout << "\nmain povikuva funkcija throwException\n";

      throwException();

      cout << "Ova ne treba da se otpecati\n";

 

   }

 

  

   catch ( exception &caughtException ) {

      cout << "\n\nIsklucok obraboten vo main\n";

 

   }

 

   cout << "Programata prodolzuva po catch vo main\n";

 

   return 0;

 

}  

 
Излезот е:
 

 

main povikuva funkcija throwException

  Funkcijata throwException frla isklucok

  Isklucok obraboten vo funkcijata throwException

  Funkcijata throwException povtorno go frla isklucokot

 

Isklucok obraboten vo main

Programata prodolzuva po catch vo main

 

Фаќање на исклучок од било кој тип

Фаќање на исклучок од било кој тип  може да се реализира на следниот начин:

catch(...) {

  cout << "nastanalo frlanje na isklucok" << endl;

}

 

Ваков catch блок (се нарекува catchall) треба да се постави после листата на сите останати catch блокови, за да не изврши нивно пресретнување. Обично се користи за ослободување на ресурси пред да го фрли повторно истиот исклучок. 

Нефатени исклучоци

Се смета дека exception handling е подобро од враќање на код за грешка бидејќи исклучоците можат да се игнорираат. Ако ниеден catch блок на по try блокот не одговара на исклучокот, исклучокот се пренесува на погорно ниво, на блок што го заградува try блокот. Процесот продолжува се додека не се најде catch кој одговара на исклучокот.

Ако на ниедно ниво нема соодветен catch блок, се повикува функцијата terminate() која е дел од стандардната библиотека. По default, terminate() ја повикува функцијата abort() која веднаш ја прекинува програмата. Не се извршува нормално завршување, што значи дека не се повикуваат деструктори за глобалните и статичките објекти. Оваа функција се повикува и ако глобален конструктор или деструктор фрли исклучок. Се повикува и ако деструктор, кој е повикан како резултат на празнење на функцискиот стек, фрли исклучок.

Функција set_terminate()

Однесувањето terminate( )на може да се промени со функцијата set_terminate( ). Со неа, terminate( ) може да се замени со корисничка функција. set_terminate( ) враќа покажувач на terminate( ) функцијата што се заменува. Функцијата што ја заменува terminate( ) не смее да има аргументи и треба да е од тип a void. Не смее да фрла исклучок.

Следниот пример илустрира  set_terminate( ).

// Upotreba na set_terminate(). Prikazuva i 'nefateni' isklucoci

#include <exception>

#include <iostream>

using namespace std;

 

void terminator() {

  cout << "Nov terminate!" << endl;

  exit(0);

}

 

 void (*old_terminate)() = set_terminate(terminator);

public:

    cout << "Mess::f()" << endl;

  }

int main() {

 

  try {

    Mess b;

Иако catch(...) би требало да го фати секој исклучок, тоа не се случува поради следното: при празнењето на стекот на try блокот за да се опслужи првиот исклучок се повикува деструкторот на Mess и заради тоа се повикува terminate( ). Бришење на објектите при фрлање на исклучок

Механизмот за исклучоци гарантира дека кога еден домен (блок) ќе заврши, тогаш за сите објекти чии конструктори комплетно се извршиле ќе бидат повикани нивните деструктори.

Следниот пример прикажува што се случува ако се фрли исклучок на средина на создавање на поле од објекти:

 

// Brisenje na objekti po exception.

#include <iostream>

using namespace std;

 

class Trace {

  static int counter;

  int objid;

public:

  Trace() {

    objid = counter++;

    cout << "konstruiram Trace #" << objid << endl;

    if(objid == 3) throw 3;

  }

  ~Trace() {

    cout << "unistuvam Trace #" << objid << endl;

  }

};

 

int Trace::counter = 0;

 

int main() {

  try {

    Trace n1;

    // Frla isklucok:

    Trace array[5];

    Trace n2;  // nema da se izvrsi.

  } catch(int i) {

    cout << "faten " << i << endl;

  }

}

 

Излезот е:

 

konstruiram Trace #0

konstruiram Trace #1

konstruiram Trace #2

konstruiram Trace #3

unistuvam Trace #2

unistuvam Trace #1

unistuvam Trace #0

faten 3

Во класата Trace се бројат објектите преку static членот counter. Секој објект го добива својот број во objid.

Во main се создава објект n1 (objid 0), и се обидува да се создаде поле од 5 Trace објекти. Бидејќи се фрла исклучок пред #3 да биде потполно креиран, n2 никогаш не се создава целосно. Од излезот се гледа дека се повикуваат деструкторите само на #0, #1 и #2. Објектот n2 не се уништува бидејќи воопшто не се ни создава.

Управување со ресурси

Кога се користат исклучоци, треба да се води сметка што настанува со алоцираните ресурси ако дојде до исклучок. Особено треба да се внимава при алоцирање на ресурси во конструктор, бидејќи ако се фрли исклучок пред да заврши конструктор, деструкторот за тој објект нема да биде повикан.

Ова најчесто се случува при алоцирање на ресурси кај покажувачи кај покажувачи (“naked pointers”).

 

// Naked pointers.

#include <iostream>

#include <cstddef>

using namespace std;

 

class Cat {

public:

  Cat() { cout << "Cat()" << endl; }

  ~Cat() { cout << "~Cat()" << endl; }

};

 

class Dog {

public:

  void* operator new(size_t sz) {

    cout << "alociram Dog" << endl;

    throw 47;

  }

  void operator delete(void* p) {

    cout << "dealociram Dog" << endl;

    delete p;

  }

};

 

class UseResources {

  Cat* bp;

  Dog* op;

public:

  UseResources(int count = 1) {

    cout << "UseResources()" << endl;

    bp = new Cat[count];

    op = new Dog;

  }

  ~UseResources() {

    cout << "~UseResources()" << endl;

    delete [] bp; // brisenje na poleto

    delete op;

  }

};

 

int main() {

  try {

    UseResources ur(3);

  } catch(int) {

    cout << "vo handler" << endl;

  }

}

 

Излезот е:

UseResources()

Cat()

Cat()

Cat()

alociram Dog

vo handler

 

Во конструкторот на UseResources се создаваат три Cat објекти, но во Dog::operator new( ), се фрла исклучок (кој симулира out-of-memory грешка). Контролата се пренесува во catch без да биде повикан деструкторот на UseResources бидејќи неговиот конструктор не завршил комплетно.Тоа значи дека трите Cat објекти кои успешно биле креирани не се уништуваат никогаш.

Вежби:

 

1.      Креирај два вектори со случаен број на елементи кој е во интервалот [1, 1000). Потоа ископирај го вториот вектор во првиот со copy. Претходно провери дали првиот вектор има големина поголема или еднаква на големината на првиот. Ако не е, да се изврши фрлање на исклучок пред да се изврши копирањето на векторот. Постапката да се повторува во циклус во кој е сместена try-catch секвенца.
