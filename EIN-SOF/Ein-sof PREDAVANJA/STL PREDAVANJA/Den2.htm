2. Параметри кај template-и

 

Една од основните особини на template-ите се нивните параметри, кои можат да бидат:

1.  вградени или кориснички типови,

2.  non-type параметри - константи познати за време на компајлирање,

3.  други template-и.

Досега разгледувавме template-и со параметри од првата категорија, кои се и најчесто застапени.  Познат пример за едноставен template е класата Stack. Stack е генеричка класа, која треба да може да сместува елементи од било кој тип. Заради тоа, тој тип се претставува преку формален тип:

template<class T> class Stack {

  T* data;

  size_t count;

public:

  void push(const T& t);

  // ...

};

 

Конкретниот тип што ќе го содржи определена инстанца на Stack е определен од параметарот T:

Stack<int> myStack;  // Stack од int елементи

 

Компајлерот потоа создава  int верзија на Stack со замена на int за T. Името на класата што се добива е Stack<int>.

 

2.1 Nontype параметри

 

Во претходните примери, class template-от Stack користеше само параметри со формални типови во своето заглавие. Можно е да се користат и nontype параметри кои ќе имаат default аргументи и се третираат како const. На пример:

template< class T, int elements>

може да се употреби да се дефинира специјализација на Stack од 100 double елементи:

Stack<double, 100> Figures;

Во дефиницијата на класата може да има приватна податочна членка која ќе декларира поле:

T stackHolder[elements];

кое ќе ги содржи елементите на складот.

 

Параметар може да дефинира и default тип:

template<class T=string>;

non-type template параметар мора да биде целобројна вредност која е позната за време на компајлирање. Stack може да се дефинира на следниот начин:

template<class T, size_t N> class Stack {

  T data[N];  // Фиксен капацитет N

  size_t count;

public:

  void push(const T& t);

  // ...

};

 

Кога се бара инстанца на template-от мора да се зададе константна вредност за N:

Stack<int, 100> myFixedStack;

            Бидејќи вредноста на N е позната за време на компајлирање, полето data може да се смести во складот, со што се заштедува времето потребно за динамичка алокација на меморија.  Името на горната класа е Stack<int, 100>. Значи, за различна вредност на N се добива единствена класа. На пример, Stack<int, 99> е различна класа од Stack<int, 100>.

Класата bitset е class template, дел од стандардната C++ библиотека. Таа користи non-type параметар кој го преставува бројот на битови кои може да ги смести објектот bitset. Следниот пример реализира генератор на случајни броеви. Тој користи bitset за евиденција на броевите кои веќе биле генерирани. Преклопен е operator( ) за да се добие синтакса слична на функциски повик.

// Urand.h

// Generator na slucajni broevi

 

#include <bitset>

#include <cstddef>

#include <cstdlib>

#include <ctime>

 

using namespace std;

 

template<size_t UpperBound> class Urand {

  bitset<UpperBound> used;

public:

  Urand() { srand(time(0)); } // Randomize

  size_t operator()(); // Generatorska funkcija

};

 

template<size_t UpperBound>

inline size_t Urand<UpperBound>::operator()() {

  if(used.count() == UpperBound)

    used.reset();  // resetiranje na bitovite

  size_t newval;

  while(used[newval = rand() % UpperBound])

    ; // povtoruvaj dodeka ne se pronajde nova slucajna vrednost

  used[newval] = true;

  return newval;

}

Броевите генерирани со Urand се единствени бидејќи bitset used ги означува сите генерирани броеви преку поставување на соодветен позициски бит. Кога сите броеви ќе се појават по еднаш, bitset се ресетира и повторно се генерираат случајни броеви во истиот простор. Следната програма е едноставен тест на објектот Urand:

 

// Tester za generator na slucajni broevi

 

#include <iostream>

#include "Urand.h"

     

 

int main() {

  Urand<10> u;

  for(int i = 0; i < 20; ++i)

    cout << u() << ' ';

} // kraj na main

 

2.2 Default вредности на параметри

 

Default аргументи можат да се зададат кај class template-и, но не и кај function template-и (компајлерот не дозволува и го игнорира default аргументот). Кога во листата на аргументи ќе се зададе default аргумент, сите параметри што следат исто така треба да имаат default аргументи. Ова е потребно поради тоа што при дефинирање на конкретен објект вредностите наведени како аргументи се придружуваат од првиот кон последниот аргумент последователно. Stack template-от со default големина може да се дефинира како:

template<class T, size_t N = 100> class Stack {

  T data[N];  // Фиксен број на елементи N

  size_t count;

public:

  void push(const T& t);

  // ...

};

 

Ако се изостави вториот аргумент при дефинирање на Stack објект, вредоста на N ќе биде 100.

Може да се зададат default вредности за сите аргументи. Во тој случај, при креирање инстанца мора да се употребат празни загради:

template<class T = int, size_t N = 100>  // Два default аргумента

class Stack {

  T data[N]; 

  size_t count;

public:

  void push(const T& t);

  // ...

};

 

Stack<> myStack;  // е исто како Stack<int, 100>

 

Иако не можат да се користат default template аргументи кај function templates, може да се користат default аргументи како аргументи на функцијата. Следниот function template пресметува сума на низа од елементи:

 

// Suma na elementi na pole

 

#include <iostream>

using namespace std;

 

 

template<class T > T sum(T* b, T* e, T init = T()) {

  while(b != e)

    init += *b++;

  return init;

}

 

int main() {

  float a[] = { 1.5, 2, 3, 4, 5, 6};

  cout << sum(a, a + sizeof a / sizeof a[0]) << endl; // 6

} // kraj na main

Третиот аргумент на sum( ) е почетната вредност на сумата на елементите. Бидејќи е изоставен, тој добива вредност T( ), која во случај на вградените типови повикува псевдо-конструктор кој врши иницијализација на 0.

 

2.3 Клучниот збор typename

Да ја разгледаме следната програма:

 

// Koristi 'typename' kako prefiks za vgnezdeni tipovi.

 

template<class T> class X {

  // Bez typename, ке se dobie greska

  typename T::id i;

public:

  void f() { i.g(); }

};

 

class Y {

public:

  class id {

  public:

    void g() {}

  };

};

 

int main() {

  X<Y> xy;

  xy.f();

}

Дефиницијата на template-от очекува класата T да има вграден идентификатор id. Ако id е статички член, тогаш на него можат да се извршуваат операции, но не и да се создаде објект од тип id. Тука id се третира како вграден тип во T. Во класата Y, id е вграден тип, но без клучниот збор typename компајлерот тоа не го знае (default параметрите на компајлерите најчесто не подразбираат дека идентификаторите претставуваат типови). Затоа, ако се користи тип во рамки на template треба да се користи клучниот збор typename.

Треба да се напомене дека typename не креира нов тип. Со:

typename Seq<T>::iterator It;

 

се декларира променлива It од тип Seq<T>::iterator. За креирање на нов тип треба да се користи typedef:

typedef typename Seq<It>::iterator It;

 

Клучниот збор typename има уште една намена. Може да се користи наместо клучниот збор class во листата на аргументи при дефиниција на template:

 

template<typename T> class X {};

  

int main() {

  X<int> x;

}

 

 

што може да претставува појасен запис.

 

 

3 Инстанцирање на template-и

 

Кога компајлерот генерира класа, функција или статичка функција членка, таа претставува инстанца на template -от.  Компајлерот генерира класа, функција или функција членка кога ќе наиде на имплицитна или експлицитна инстанца на template.

Кога се врши инстанцирање на class template, се генерира кодот на дефиницијата на класата заедно со сите функции членки што се повикани во програмата. Само функциите членки што се повикани во програмата се генерираат. Ваквиот принцип е корисен бидејќи спречува непотребно зголемување на генерираниот изворен код.

1. Следниот пример илустрира инстанца на class template.

template <class T>

class Z

{

  public:

Z() {} ;

~Z() {} ;

void f(){} ;

void g(){} ;

} ;

 

int main()

{

Z<int> zi ;   //implicitna instanca na klasa Z<int>

Z<float> zf ; // implicitna instanca na klasa Z<float>

return 0 ;

}

 

2. Во следната програма се користат функциите членки Z<T>::f() и Z<T>::g().

template <class T>

class Z

{

  public:

   Z() {} ;

   ~Z() {} ;

   void f(){} ;

   void g(){} ;

} ;

 

int main()

{

   Z<int> zi ; // implicitna instanca generira klasa Z<int>

   zi.f() ;    //i funkcija clenka Z<int>::f()

   Z<float> zf ; // implicitna instanca generira klasa Z<float>

   zf.g() ;      // i funkcija clenka Z<float>::g()

   return 0 ;

}

 
Во овој случај, покрај генерирањето на класите Z<int> и Z<float>, со конструктори и деструктори, компајлерот исто така ги генерира и дефинициите на Z<int>::f() и Z<float>::g(), бидејќи тие се повикуваат во main. Компајлерот не генерира дефиниции на функции членки кои не е потребно инстанцирање.  Во овој пример, компајлерот не генерира код за Z<int>::g() и Z<float>::f(), бидејќи тие воопшто не се повикуваат.

3. Во следниот пример компајлерот воопшто нема да генерира класа:

#include <iostream>

template <class T>

class Z

{

  public:

   Z() {cout <<”Konstruktor\n”;}

   ~Z() { cout <<”Destruktor\n”;}

   void f(){} ;

   void g(){} ;

} ;

 

int main()

{

   Z<int>* p_zi ; //Instanciranje na Z<int> ne e potrebno

   Z<float>* p_zf ; //Instanciranje na Z<float> ne e potrebno

   return 0 ;

}

 

Дека не се генерира код може да се види според тоа што не доаѓа до извршување ниту на конструкторот ниту на деструкторот. Нема потреба од дефиниции бидејќи се декларирани само покажувачи.

4. Следниот пример илустрира имплицитно инстанцирање на function template.

//max vrakja maksimum na dva elementi

template <class T>

T max(T a, T b)

{

    return a > b ? a : b ;

}

void main()

{

   int I ;

   I = max(10, 15) ; //implicitna instanca na max(int, int)

   char c ;

   c = max('k', 's') ; //implicitna instanca na max(char, char)

}

 

Компајлерот генерира функции max(int, int) и max(char, char) користејќи го  template-от max.

5. Ако е потребна инстанца на class template, а тој е деклариран, но не и дефиниран, компајлерот VC6.0 генерира грешка C2079.

template <class T> class X ;

 

int main()

{

   X<int> xi ; //error C2079: 'xi' koristi nedefinirana klasa 'X<int>'

   return 0 ;

}

 

6. Со тоа што се генерира само кодот на функциите што се користат се спречува непотребно зголемување на изворниот код. Во спротивно, генерирањето на сите функции членки при инстанцирање на class template во голема мера ќе го ограничи бројот на употребени template-и во една програма. 

 

//

 

class X {

public:

  void f() {}

};

 

class Y {

public:

  void g() {}

};

 

template<typename T> class Z {

  T t;

public:

  void a() { t.f(); }

  void b() { t.g(); }

};

 

int main() {

  Z<X> zx;

  zx.a(); // ne se kreira Z<X>::b()

  Z<Y> zy;

  zy.b(); // ne se kreira Z<Y>::a()

} //

 

template-от Z овозможува употреба на членките на Т  f( ) и g( ) . Но, од тоа што овој код се компајлира се гледа дека се генерира само Z<X>::a( ) кога е експлицитно повикан за zx. Ако беше генериран и Z<X>::b( ) ќе се јавеше грешка при компајлирање бидејќи Z<X>::b( ) врши повик на X::g( ), која не постои. Слично, повикот на zy.b( ) не ја генерира Z<Y>::a( ). Поради тоа, Z template-от може да се користи и со X и со Y. 
 
 
 
 
 

Вежби:

 

 

1.      Напиши function template кој има еден параметризиран тип (T) и прифаќа четири аргументи: поле од тип T, почетен индекс, краен индекс и опциона, почетна вредност. Функцијата трба да ја врати сумата на елементите во полето зададени со почетниот и крајниот индекс. За почетната вредност да се употреби конструкторот на T.

2.      Креирај класа Kvadrat со податоци strana и класа Pravoagolnik со податоци sirina и dolzina.  Напиши function template sortirajPole со кој се врши сортирање на поле и function template pecatiPole со кој се врши печатење на поле. Во main креирај две полиња од квадрати и праваголници, прикажи ги со pecatiPole, сортирај ги според плоштина со sortirajPole и повторно отпечати ги.

3.      Една фирма води евиденција за лица кои можат да бидат вработени или клиенти. За секое лице се чуваат следните податоци:  imeiprezime, mat_br, vozrast, adresa, kod. За вработените kod има целобројна вредност, а за клиентите е низа од знаци.

Креирај class template Lica во која ќе се сместуваат податоците за лица. Податоците кои се низи од знаци да се внесуваат преку динамичка алокација на меморија. Креирај две полиња од објекти, Vraboteni и Klienti. За печатење на двете полиња употреби ја pecatiPole од претходната задача.
