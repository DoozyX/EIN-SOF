C++ Stream Input/Output

C++ применува type safe влез/излез при што операциите се извршуваат во согласност со типот на податоците. Покрај тоа што се дефинирани функции членки за многу вградени типови, постои можност за нивно проширување за употреба со кориснички типови.

Влезот и излезот се одвива преку потоци (streams) кои се всушност низи од бајти. Кај влезните операции потокот тече од уред кон главната меморија, а во излезните во спротивната насока. Апликациите го определуваат значењето на низата од бајти.

Може да се врши неформатиран В/И (low level) во кој низата се третира бајт по бајт, или форматиран В/И (high level) во кој бајтите се групираат во целини кои претставуваат податочен тип (int, float, кориснички тип итн.)

Во минатото C++ библиотеките за работа со потоци овозможуваа само работа со char со кои може да се претстави само ограничени множество од знаци. Сега C++ вклучува стандардни библиотеки за работа со потоци со кои може да се врши В/И со Unicode знаци. Вклучен е додатен тип wchar_t со кој може да се претставуваат Unicode знаци. Класите за потоци се редизајнирани како class template-и со посебни специјализации за char и wchar_t.

Заглавни датотеки за работа со потоци се <iostream>, во која се декларирани основните сервиси за В/И, <iomanip> содржи манипулатори за форматиран В/И и <fstream> содржи сервиси за процесирање на датотеки.

За извршување на В/И операции се преклопени операторите << - stream-insertion operator и  >> - stream-extraction operator. Тие се користат со стандардните објекти cin, cout, cerr и clog.

Класи и објекти на C++ Stream Input/Output

Библиотеката iostream содржи множество template-и за В/И операции. basic_istream е class template кој поддржува влезни операции кај потоци, basic_ostream е class template кој поддржува излезни операции кај потоци, а basic_iostream поддржува влезни и излезни операции. Секој template има преддефинирани специјализации за char В/И.

basic_istream и basic_ostream наследуваат од basic_ios. basic_iostream наследува од basic_istream и basic_ostream.

За процесирање на датотеки се користат basic_ifstream (за влез), basic_ofstream (за излез) и basic_fstream (за влез и излез).

ios_base ги содржи сите декларации заеднички за сите потоци.

Печатење на char*

Заради тоа што во C++ се врши автоматско определување на типот, не може директно да се отпечати вредност на покажувачот char*. << е преклопен да ја печати вредноста на char* како стринг. За да се отпечати адреса, треба да се примени конверзија:

char *zbor=”tekst”;

cout << static_cast<void *>(zbor);

Печатење со функција членка put

cout.put(‘A’); - печати А

cout.put(‘A’).put(‘\n’); - може да се користи каскадно

cout.put(65); - печати А, аргументот се третира како ASCII код

Влез

get – без аргументи чита еден знак од влезен поток и го враќа како резултат на функциски повик. Верзијата со два аргументи има поле од знаци како прв аргумент и максимална големина на полето. Постои и трета верзија во која има трет аргумент, знак за одвојување, кој има предефинирана вредност ‘\n’. Оваа верзија завршува со читање кога ќе се надмине лимитот на полето или кога ќе се појави знак за одвојување. Знакот за одвојување не се сместува во полето, но останува во влезниот поток. Кога се врши читање со cin >> се прекинува вчитувањето при појава на првиот бланко знак.

getline функционира слично, со таа разлика што знакот за одвојување не го сместува во полето, но го отстранува од влезниот поток.

ignore чита и отфрла даден број знаци или завршува на знак за одвојување.

putback знак кој претходно е прочитан со get го враќа назад во влезниот поток. peek го враќа следниот знак во потокот, но не го отстранува од него.

//  get, getline, put, eof.

#include <iostream>

 

using std::cout;

using std::cin;

using std::endl;

 

int main()

{

   int character; // koristi int za pretstavuvanje na  EOF

   const int SIZE = 80;

   char buffer1[ SIZE ];

   char buffer2[ SIZE ];

 

   cout << "Vnesi recenica:" << endl;

   cin >> buffer1;

 

 

   cout << "\nStringot procitan so cin e:" << endl

        << buffer1 << endl << endl;

  

   cin.get( buffer2, SIZE );

 

   cout << "Stringot procitan so cin.get e:" << endl

        << buffer2 << endl;

 

   cin.ignore();//Potrebno za da se otstrani '\n' od vlezniot potok

   cout << "Vnesi recenica:" << endl;

   cin.getline( buffer1, SIZE );

 

 

   cout << "\nRecenicata e:" << endl << buffer1 << endl;

 

   cout << "Pred vnes cin.eof() e " << cin.eof() << endl

        << "Vnesi recenica, end-of-file za kraj:" << endl;

 

   //Vrednosta na end-of-file zavisi od OS

   //<Ctrl>-Z na PC, <Ctrl>-D na Linux i Mac

   //Najcesto e potrebno da se pritisne <Enter> po <Ctrl>-Z

 

  

   while ( ( character = cin.get() ) != EOF )

      cout.put( character );

 

  

   cout << "\nEOF na ovoj sistem e: " << character << endl;

   cout << "Po vnes, cin.eof() e " << cin.eof() << endl;

   return 0;

 

}

Неформатиран влез

Неформатиран влез и излез се врши со функциите read и write на istream и ostream соодветно. Со нив се врши влез и излез на низа од знаци кои немаат никаков формат.

cout.write(buffer, 10);

ги печати првите 10 знаци од полето, вклучително со нула (не ја смета за ознака за крај).

Со read се врши читање на зададениот број на бајти. Ако се прочитани помалку отколку што е аргументот, се поставува битот failbit. gcount го враќа бројот на внесени знаци.

Манипулатори

Манипулаторите вршат разни видови на форматирање на потоците.

Преддефинирано, броевите се интерпретираат во декаден систем. Со dec, oct, hex и setbase може да се дефинираат претставувања со различни бројни системи. За да се користат, потребно е да се вклучи <iomanip>.

Записот на децималните броеви може да се контролира со setprecision или со precision, функцијата членка на ios_base.

За дефинирање на број на знаци со кој треба да се прикаже вредност се користи функцијата членка на ios_base, width и манипулаторот setw. Ако вредноста што се прикажува е помала од дефинираната, се користат онолку места колку што е потребно да се прикаже целата вредност.

// hex, oct, dec, setbase, precision, setprecision.

#include <iostream>

 

using std::cout;

using std::cin;

using std::endl;

using std::hex;

using std::dec;

using std::oct;

 

#include <iomanip>

 

#include <cmath>

using std::setprecision;

using std::setbase;

using std::fixed;

 

int main()

{

   int number;

 

   cout << "Vnesi dekaden broj: ";

   cin >> number;

 

  

   cout <<" hex: " << hex << number << " oct: "

        << oct << number << setbase( 10 ) << " dec: "

        << number << endl;

 

      double root2 = sqrt( 2.0 ); // calculate square root of 2

   int places;

 

   cout << "Koren od 2 so preciznost 0-4." << endl

        << "postavena so ios_base::precision \n";

 

   cout << fixed; // use fixed precision

 

   for ( places = 0; places <= 4; places++ ) {

      cout.precision( places );

      cout << root2 << endl;

   }

 

   cout << "\nPreciznost postavena so setprecision\n " ;

 

   for ( places = 0; places <= 4; places++ )

      cout << setprecision( places ) << root2 << endl;

   return 0;

}

Манипулаторите го поставуваат форматот на излезниот поток. И припаѓаат на класата ios_base

  • boolalpha - прикажува true и false за bool вредности. Се ресетира со noboolalpha.
  • dec - декаден формат.
  • fixed - фиксна точка.
  • hex - хексадекаден формат.
  • internal - лево порамнување на знак и десно порамнување на вредност.
  • left - лево порамнување.
  • oct - октален формат.
  • right - десно порамнување.
  • scientific - експоненцијален формат.
  • showbase - прикажува бројна база како префикс. Се ресетира со noshowbase.
  • showpoint - безусловно прикажување на децимална точка. Се ресетира со noshowpoint.
  • showpos - прикажува знак и кај ненегативни вредности. Се ресетира со noshowpos.
  • skipws - прескокнување на водечки празни места. Се ресетира со noskipws.
  •  
  • uppercase - прикажува големи букви кај бројни вредности (пр. 4.5Е+01, 5BC12). Се ресетира со nouppercase.

Работа со датотеки

Датотеките во C++ се третираат како низи од бајти. Датотеките може да завршуваат со end-of-file маркер или големината да им биде оределена со специфициран број на бајти кај структурирана датотека.

Кога се отвора датотека, се создава објект и со тој објект се поврзува поток. Потокот ја обезбедува комуникацијата помеѓу програмата и датотеката.

Датотеките се отвораат преку креирање на објекти од специјализации на basic_ifstream, basic_ofstream или basic_fstream. Секој class template има специјализација која овозможува char В/И. fstream содржи typedefs за овие специјализации. На пример, typedef ifstream овозможува влез од датотека,  typedef ofstream излез, а typedef fstream влез и излез.

Креирање и читање на секвенцијална датотека

Датотеките немаат структура, не постои дефиниран поим запис. Од програмерот зависи како ќе ја структурира содржината на датотеката.

Следниот пример содржи креирање и читање на датотека во која содржината се структурира како запис кој содржи сметка, име и состојба.

// Rabota so sekvencijalna datoteka.

#include <iostream>

 

using std::cout;

using std::cin;

using std::ios;

using std::cerr;

using std::endl;

 

using std::left;

using std::right;

using std::fixed;

using std::showpoint;

 

#include <fstream>

 

using std::ifstream;

using std::ofstream;

 

#include <iomanip>

 

using std::setw;

using std::setprecision;

 

#include <cstdlib> 

 

void outputLine( int, const char * const, double );

 

int main()

{

   // konstruktorot na ofstream otvora datoteka

   ofstream outClientFile( "clients.dat", ios::out );

 

  

   if ( !outClientFile ) {  // preklopen ! operator za ofstream

      cerr << "Datotekata ne moze da se otvori" << endl;

      exit( 1 );

 

   }

 

   cout << "Vnesi broj na smetka, ime, i sostojba." << endl

        << "Vnesi end-of-file za kraj na vnes.\n? ";

 

   int account;

   char name[ 30 ];

   double balance;

 

  

   while ( cin >> account >> name >> balance ) {

      outClientFile << account << ' ' << name << ' ' << balance

                    << endl;

      cout << "? ";

 

   }

      //zatvoranje na datotekata

      outClientFile.close();

 

   // konstruktorot na ifstream ja otvora istata datoteka za citanje

   ifstream inClientFile( "clients.dat", ios::in );

 

  

   if ( !inClientFile ) {

      cerr << "Datotekata ne moze da se otvori" << endl;

      exit( 1 );

 

   }

 

   cout << left << setw( 10 ) << "Smetka" << setw( 13 )

        << "Ime" << "Sostojba" << endl << fixed << showpoint;

 

   // prikazi go sekoj zapis

   while ( inClientFile >> account >> name >> balance )

      outputLine( account, name, balance );

 

   return 0;  // destruktorot na ofstream ja zatvora datotekata

} // kraj na main

 

 

void outputLine( int account, const char * const name,

   double balance )

{

   cout << left << setw( 10 ) << account << setw( 13 ) << name

        << setw( 7 ) << setprecision( 2 ) << right << balance

        << endl;

 

} // kraj na outputLine

typedef basic_ios<char, char_traits<char> > ios;

Тој е всушност синоним за template class basic_ios, специјализиран за char.

Можно е репозиционирање во секвенцијална датотека преку методите seekg кај istream и seekp кај ostream. Со tellg и tellp се враќа тековната позиција на покажувачите на get и put соодветно.

 

Примери:

 

FObject.seekg(n); - позицоинирање на n-тиот бајт

FObject.seekg(n, ios::cur); - движење за n бајти напред

FObject.seekg(n, ios::end); - движење за n бајти назад од крајот на датотеката

FObject.seekg(0, ios::end); - преместување на крај

 

Ако читањето дојде до крај на датотека, FObject.eof() враќа true, за да се почне повторно со читање, претходно треба да се ресетира eof со FObject.clear().

 

Ажурирање на секвенцијални датотеки

 

Ажурирањето на секвенцијалните датотеки значи оштетување на другите податоци во датотеката. Записите варираат во големина и промена на еден запис кој е поголем од претходниот, значи промена и на следниот запис по него. Еден начин да се ажурира секвенцијална датотека е да се препише содржината пред записот кој се менува во друга датотека, да се допише неговата нова вредност и потоа и остатокот од датотеката.

 

Вежби:

 

1.      Табелирај ја вредноста на i, i2,  и за вредности на i=1,2,... n. Ведностите да бидат прикажани во колони со ширина m (за децималните вредности да се дефинира соодветна прецизност). Вредностите на n и m треба да се внесат на почетокот на програмата.

2.      Напиши програма со која сите зборови од текстуална датотека со големина помала од 10 знаци ќе ги запише во друга текстуална датотека.
