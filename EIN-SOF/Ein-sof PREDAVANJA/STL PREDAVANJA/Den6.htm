4.6 Асоцијативни контејнери

 

 

Асоцијативните контејнери од STL овозможуваат директен пристап до елементи преку клучеви кои секогаш се складирани сортирани. Во оваа група спаѓаат: multiset, set, multimap и map. multiset и set обезбедуваат операции за манипулирање со множества од вредности, при што самите вредности претставуваат клучеви. Разликата меѓу нив е тоа што multiset дозволува постоење на исти вредности, додека set не дозволува. Кај multimap и map секоја вредност е асоцирана со клуч. multimap дозволува исти вредности на клучевите, а map не. Покрај функциит заеднички за сите контејнери, асоцијативните контејнери поддржуваат и find, lower_bound, upper_bound и count.

 

4.6.1 Asocijativen контејнер multiset

multiset обезбедува брзо складирање и пристап до елементи на множество од вредности преку клучеви. Подредувањето на елементите е определено преку objekt компаратор (comparator function object). На пример, кај int multiset, елементите можат да се сортираат преку употреба на компараторот less<int>. Податочните типови на клучевите кај сите асоцијативни контејнери мора да поддржуваат споредба преку операторот <. multiset поддржува bidirectional итератори.

Следната програма илустрира употреба на multiset од цели броеви сортирани во растечки редослед. За да се користи multiset треба да се вклучи заглавието <set>.

 

 

// Testiranje na funkcii  na multiset

 

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <set>  //  class-template definicija na multiset

 

// definicija na int multiset type od ime ims

typedef std::multiset< int, std::less< int > > ims;

 

 

#include <algorithm> 

 

 

int main()

{

   const int SIZE = 10;

   int a[ SIZE ] = { 7, 22, 9, 1, 18, 30, 100, 22, 85, 13 };

 

   ims intMultiset; 

   std::ostream_iterator< int > output( cout, " " );

 

   cout << "Ima " << intMultiset.count( 15 )

        << " pojavuvanja na 15 vo intMultiset\n";

  

   intMultiset.insert( 15 );  // vmetni 15 vo intMultiset

   intMultiset.insert( 15 );  // vmetni 15 vo intMultiset

 

   cout << "Po vmetnuvanjeto, ima "

        << intMultiset.count( 15 )

        << " vrednosti 15 vo intMultiset\n\n";

 

   // iterator so koj ne moze da se menuvaat vrednostite

   ims::const_iterator result; 

 

 

   // pronajdi 15 vo intMultiset; find vrakja iterator

   result = intMultiset.find( 15 ); 

 

   if ( result != intMultiset.end() ) // ako iteratorot ne e na kraj

      cout << "Pronajdena vrednost 15\n";     // pronajdena e vrednost 15

  

    // pronajdi 20 vo intMultiset; find vrakja iterator

   result = intMultiset.find( 20 );

 

   if ( result == intMultiset.end() )    // ==true

      cout << "Ne e pronajdena vrednost 20\n"; //ne e najdena 20

 

   // vmetni elementi od poleto array vo intMultiset

   intMultiset.insert( a, a + SIZE );

 

   cout << "\nPo vmetnuvanjeto, intMultiset sodrzi:\n";

   std::copy( intMultiset.begin(), intMultiset.end(), output );

 

  

// najdi go prvoto i poslednoto pojavuvanje na 22 vo intMultiset

   cout << "\n\nPrvo pojavuvanje na 22: "

        << *( intMultiset.lower_bound( 22 ) );

   cout << "\nPosledno pojavuvanje na 22: "

        << *( intMultiset.upper_bound( 22 ) );

 

   // p e par od const_iterators

   std::pair< ims::const_iterator, ims::const_iterator > p;

 

   // opredeli prvoto i poslednoto pojavuvanje na 22 preku 

   p = intMultiset.equal_range( 22 );

 

   cout << "\n\nequal_range na 22:"

        << "\n   Prvo pojavuvanje: " << *( p.first )

        << "\n   Posledno pojavuvanje: " << *( p.second );

 

   cout << endl;

 

 

   return 0;

 

} // kraj na main

 

 

Резултатот од извршување на програмата е:

Ima 0 pojavuvanja na 15 vo intMultiset

Po vmetnuvanjeto, ima 2 vrednosti 15 vo intMultiset

Pronajdena vrednost 15

Ne e pronajdena vrednost 20

 

Po vmetnuvanjeto, intMultiset sodrzi:

1 7 9 13 15 15 18 22 22 30 85 100

 

Prvo pojavuvanje na 22: 22

Posledno pojavuvanje na 22: 30

 

equal_range na 22:

   Prvo pojavuvanje: 22

   Posledno pojavuvanje: 30

lower_bound го пронаоѓа првото појавување на елементот со вредноста зададена како аргумент. upper_bound го пронаоѓа првиот елемент после последното појавување на елементот со вредноста зададена како аргумент.

cout << "\n\nPrvo pojavuvanje na 22: "

        << *( intMultiset.lower_bound( 22 ) );

   cout << "\nPosledno pojavuvanje na 22: "

        << *( intMultiset.upper_bound( 22 ) );

Двете функции враќаат iterator или const_iterator кој покажуваат на соодветните елементи.

Објектите од класата pair се користат за дефинирање на парови од објекти.

   // p e par od const_iterators

   std::pair< ims::const_iterator, ims::const_iterator > p;

 

   // opredeli prvoto i poslednoto pojavuvanje na 22 preku 

   p = intMultiset.equal_range( 22 );

Во примерот содржината на p се два итератори кои ги враќа equal_range. Таа враќа итератори на првото и последното појавување на елементот со вредноста 22. Тие се резултат на извршувањето на lower_bound и upper_bound. До нив се пристапува преку p.first и p.second (first и second се јавни статички членови на класата pair).

4.6.2 Asocijativen контејнер set

 

Контејнерот set е наменет за брзо складирање и пристап до множество од единствени вредности. Имплементацијата на set е идентична како таа на multiset, освен што не се дозволени исти вредности на елементи. set поддржува bidirectional итератори, но не и random-access.

 

Следниот пример илустрира употреба на set од double елементи. За да се користи set треба да се вклучи заглавието <set>.

 

 

// Testiranje na funkcii  na set

 

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <set>

 

 

// definicija na double set type pod ime double_set

typedef std::set< double, std::less< double > > double_set;

 

#include <algorithm>

 

int main()

{

   const int SIZE = 5;

   double a[ SIZE ] = { 2.1, 4.2, 9.5, 2.1, 3.7 };  

 

   double_set doubleSet( a, a + SIZE );

   std::ostream_iterator< double > output( cout, " " );

 

   cout << "doubleSet sodrzi: ";

   std::copy( doubleSet.begin(), doubleSet.end(), output );

 

   // p e pair sto sodrzi const_iterator i bool

   std::pair< double_set::const_iterator, bool > p;

 

   // vmetni 13.8 vo doubleSet; povratnata vrednost smesti ja vo pair p

   // p.first e lokacijata na 13.8 vo doubleSet 

   // p.second dali 13.8 e vmetnata

   p = doubleSet.insert( 13.8 ); // 13.8 ja nema prethodno vo mnoзestvoto

 

   cout << "\n\n" << *( p.first )

        << ( p.second ? " e" : " ne e" ) << " vmetnata";

 

   cout << "\ndoubleSet sodrzi: ";

   std::copy( doubleSet.begin(), doubleSet.end(), output );

 

   // vmetni 9.5 vo doubleSet

   p = doubleSet.insert( 9.5 );  // 9.5 veke ja ima vo mnosestvoto

 

   cout << "\n\n" << *( p.first )

        << ( p.second ? " e" : " ne e" ) << " vmetnata";

  

   cout << "\ndoubleSet sodrzi: ";

   std::copy( doubleSet.begin(), doubleSet.end(), output );

 

   cout << endl;

 

   return 0;

 

} // kraj na main

 

 Резултатот од извршување на програмата е:

doubleSet sodrzi: 2.1 3.7 4.2 9.5

 

13.8 e vmetnata

doubleSet sodrzi: 2.1 3.7 4.2 9.5 13.8

 

9.5 ne e vmetnata

doubleSet sodrzi: 2.1 3.7 4.2 9.5 13.8

 

 

   p = doubleSet.insert( 13.8 ); 

 

Парот p добива вредност на const_iterator и bool кои ги враќа insert. Вториот аргумент има вредност true ако се вметне вредноста, а false ако не се вметне вредноста бидејќи таа веќе ја има во множеството.

 

4.6.3 Asocijativen контејнер multimap

 

Контејнерот multimap е наменет за брзо складирање и пристап до множество од вредности и клучеви. Кога се вметнува во multimap и map се користи објект pair што содржи клуч и вредност. Подредувањето на елементите е определено преку објект компаратор (comparator function object). На пример, кај int multimap, елементите можат да се сортираат преку употреба на кoмпараторот less<int>. Дозволена е употреба на елементи со исти вредности и повеќе вредности можат да бидат асоцирани со еден клуч. Ова се нарекува релација еден-према-многу. multimap поддржува bidirectional итератори, но не и random-access.

Следнaта програма илустрира употреба на multimap. За да се користи multimap треба да се вклучи заглавието <map>.

 

// Test programa za multimap.

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <map>  //definicija na map class-template

 

// definicija na multimap type pod ime mmid

typedef std::multimap< int, double, std::less< int > > mmid;

 

int main()

{

   mmid pairs;

 

   cout << "Ima " << pairs.count( 15 )

        << " parovi so kluc 15 vo pairs\n";

 

   // vmetni dva value_type objekti vo pairs

   pairs.insert( mmid::value_type( 15, 2.7 ) );

   pairs.insert( mmid::value_type( 15, 99.3 ) );

  

   cout << "Po vmetnuvanjeto ima "

        << pairs.count( 15 )

        << " parovi so kluc 15\n\n";

  

   // vmetni pet value_type objekti vo pairs

   pairs.insert( mmid::value_type( 30, 111.11 ) );

   pairs.insert( mmid::value_type( 10, 22.22 ) );

   pairs.insert( mmid::value_type( 25, 33.333 ) );

   pairs.insert( mmid::value_type( 20, 9.345 ) );

   pairs.insert( mmid::value_type( 5, 77.54 ) );

  

   //vmetnuvanje na objekt od tip pair

   std::pair< int, double > p;

   p.first=5; p.second=11.1;

   pairs.insert( p);

 

   //definiranje na tip na elementi kako pair i vmetnuvanje na

   //element preku konstruktor

   typedef std::pair< int, double > El;

   pairs.insert(El(5,10.5));

 

   cout << "Multimap pairs sodrzi:\nKluc\tVrednost\n";

  

   // koristi const_iterator za dvizenje niz elementite

   for ( mmid::const_iterator iter = pairs.begin();

         iter != pairs.end(); ++iter )

      cout << iter->first << '\t'

           << iter->second << '\n';

 

   cout << endl;

 

   return 0;

 

} // kraj na main

Po vmetnuvanjeto ima 2 parovi so kluc 15

Kluc    Vrednost

5       11.1

5       10.5

10      22.22

15      2.7

15      99.3

20      9.345

25      33.333

30      111.11

 

 

// definicija na multimap type pod ime mmid

typedef std::multimap< int, double, std::less< int > > mmid;

 

Во примерот се дефинира тип кој е multimap во кој типот на клучевите е int, асоцираните вредности се double, а елементите се сортирани во растечки редослед.

Со:

   pairs.insert( mmid::value_type( 15, 2.7 ) );

се врши вметнување пар од целоброен клуч со вредност 15 и double вредност 2.7. Овој пар го враќа во вид на објект pair функцијата value_type.

 

Елемент може да се вметне и преку дефинирање на соодветен објект од тип pair чиј прв елемент е од тип на клучот, а втор од тип на вредноста на multimap.   

 

   //vmetnuvanje na objekt od tip pair

   std::pair< int, double > p;

   p.first=5; p.second=11.1;

   pairs.insert( p);

 

   //definiranje na tip na elementi kako pair i vmetnuvanje na

   //element preku konstruktor

   typedef std::pair< int, double > El;

   pairs.insert(El(5,10.5));

 

За движење низ елементите се употребува const_iterator iter. Се прикажуваат и клучевите и вредностите.

 

   for ( mmid::const_iterator iter = pairs.begin();

         iter != pairs.end(); ++iter )

      cout << iter->first << '\t'

           << iter->second << '\n';

 

 

4.6.4 Asocijativen контејнер map

 

Контејнерот map е наменет за брзо складирање и пристап до множество од вредности и клучеви. Кај map не се дозволени исти вредности на елементите. Кај него постои пресликување еден-према-еден. Честопати map се нарекува и асоцијативно поле. За пристап до елементите може да се користи subscript операторот [].

Примерот во продолжение илустрира употреба на map. За да се користи map треба да се вклучи заглавието <map>.

 

 

// Test programa za map .

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <map>  // definicija na map class-template

 

// definicija na map type pod ime mid

typedef std::map< int, double, std::less< int > > mid;

 

int main()

{

   mid pairs;

 

   // vmetni osum value_type objekti vo pairs

   pairs.insert( mid::value_type( 15, 2.7 ) );

   pairs.insert( mid::value_type( 30, 111.11 ) );

   pairs.insert( mid::value_type( 5, 1010.1 ) );

   pairs.insert( mid::value_type( 10, 22.22 ) );

   pairs.insert( mid::value_type( 25, 33.333 ) );

   pairs.insert( mid::value_type( 5, 77.54 ) ); // duplikat, se ignorira

   pairs.insert( mid::value_type( 20, 9.345 ) );

   pairs.insert( mid::value_type( 15, 99.3 ) ); // duplikat, se ignorira

 

   cout << "pairs sodrzi:\nKluc\tVrednost\n";

  

   // koristi const_iterator za dvizenje niz elementite na pairs

   for ( mid::const_iterator iter = pairs.begin();

         iter != pairs.end(); ++iter )

      cout << iter->first << '\t'

           << iter->second << '\n';

 

   // koristi subscript operator za promena na vrednosta za klucot 25

   pairs[ 25 ] = 9999.99; 

  

   // koristi subscript operator za promena na vrednosta za klucot 40

   pairs[ 40 ] = 8765.43; 

  

   cout << "\nPo subscript operaciite, pairs sodrzi:"

        << "\nKluc\tVrednost\n";

  

   for ( mid::const_iterator iter2 = pairs.begin();

         iter2 != pairs.end(); ++iter2 )

      cout << iter2->first << '\t'

           << iter2->second << '\n';

 

   cout << endl;

 

   return 0;

 

} // kraj na main

4.7 Адаптери

 

Во STL постојат три контејнери кои се нарекуваат адаптери: stack, queue и priority_queue. Адаптерите не се првокласни контејнери и не поддржуваат итератори. Со адаптерите може да се одбере соодветна податочна структура  (underlying data structure). Сите адаптери содржат функции членки push и pop кои вршат вметнување и бришење на елемент од податочната структура која ја користат.

4.7.1 Адаптер stack

 

stack овозможува вметнување и бришење на елемент на еден крај од податочната структура која ја користи (underlying data structure). stack може да биде имплементиран со било кој од секвенцијалните контејнери: vector, list или deque. По default, stack се имплементира со deque.

Во примерот се креираат три stack објекти, секој со еден од секвенцијалните контејнери. Операции кои ги поддржува stack се push (ја повикува push_back од контејнерот што го користи), pop(ја повикува pop_back од контејнерот што го користи), empty и size.

За да се користи stack треба да се вклучи заглавието < stack >.

 

// Test za STL stack.

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <stack>   // definicija na stack adapter

#include <vector>  // definicija na vector class-template

#include <list>    // definicija na list class-template

 

// prototip na popElements function-template 

template< class T >

void popElements( T &stackRef );

 

int main()

{

   // stack so default underlying deque

   std::stack< int > intDequeStack;  

 

   // stack so underlying vector

   std::stack< int, std::vector< int > > intVectorStack;

 

   // stack so underlying list

   std::stack< int, std::list< int > > intListStack;

 

   // push vrednosti 0-9 na sekoj stack

   for ( int i = 0; i < 10; ++i ) {

      intDequeStack.push( i );

      intVectorStack.push( i );

      intListStack.push( i );

 

   }

 

   // prikazi gi i otstrani gi vrednostite od sekoj stack

   cout << "Pop od intDequeStack: ";

   popElements( intDequeStack );

   cout << "\nPop od intVectorStack: ";

   popElements( intVectorStack );

   cout << "\nPop od intListStack: ";

   popElements( intListStack );

 

   cout << endl;

 

   return 0;

 

} // kraj na main

 

// pop na elementite od objekt stack na koj pokazuva stackRef

template< class T >

void popElements( T &stackRef )

{

   while ( !stackRef.empty() ) {

      cout << stackRef.top() << ' ';  // prikazi go elementot na vrvot

      stackRef.pop();                 // otstrani go elementot na vrvot

 

   }

 

} // kraj na funkcijata popElements

Излез од програмата:

Pop od intDequeStack: 9 8 7 6 5 4 3 2 1 0

Pop od intVectorStack: 9 8 7 6 5 4 3 2 1 0

Pop od intListStack: 9 8 7 6 5 4 3 2 1 0

 

4.7.2 Адаптер queue

 

queue овозможува вметнување на крај и бришење на елементи на почеток од податочната структура која ја користи. queue може да биде имплементиран со list или deque. По default, queue се имплементира со deque.

Операции кои ги поддржува queue се push за додавање на елемент на крај (ја повикува push_back од контејнерот што го користи), pop за отстранување на елемент на почетокот (ја повикува pop_front од контејнерот што го користи), back, која враќа референца на последниот елемент (ја повикува back од контејнерот што го користи), empty и size.

За да се користи queue треба да се вклучи заглавието < queue >.

Во примерот се креира double queue објект.

 

// Test programa za STL adapter queue

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <queue>  // definicija na adapter queue 

 

int main()

{

   std::queue< double > values;

  

   // push na elementi na values

   values.push( 3.2 );

   values.push( 9.8 );

   values.push( 5.4 );

 

   cout << "Pop na elementi od redicata values: ";

  

   while ( !values.empty() ) {

      cout << values.front() << ' ';  // prikazi go elementot na pocetokot

      values.pop();                   // otstrani go

 

   }

 

   cout << endl;

 

   return 0;

 

} // kraj na main

 

Излез од програмата:

Pop na elementi od redicata values: 3.2 9.8 5.4

 

 

4.7.3 Адаптер priority­_queue

 

priority_queue овозможува сортирано вметнување и бришење на елементи на почеток од податочната структура која ја користи. priority_queue може да биде имплементиран со vector или deque. По default, priority_queue се имплементира со vector. Кога се додаваат елементи во priority_queue се врши сортирање според приоритет. Елементот со најголем приоритет (односно, елементот со најголема вредност)  ќе биде отстранет од редицата. За споредување се користи објектот less<T>, но може да се примени и друг компаратор.

Операции кои ги поддржува queue се push за додавање на елемент на соодветна локација определена според приоритетот (ја повикува push_back од контејнерот што го користи, а потоа користи heapsort), pop за отстранување на елементот со најголем приоритет (ја повикува pop_back од контејнерот што го користи), back, која враќа референца на последниот елемент (ја повикува back од контејнерот што го користи), top враќа референца на првиот елемент (ја повикува front од контејнерот што го користи), empty (ја повикува empty од контејнерот што го користи) и size (ја повикува size од контејнерот што го користи).

За да се користи priority_queue треба да се вклучи заглавието < queue >.

Во примерот се креира double priority_queue објект.

// Test programa za STL priority_queue.

 

#include <iostream>

 

using std::cout;

using std::endl;

 

#include <queue>  // definicija na adapter priority_queue

 

int main()

{

   std::priority_queue< double > priorities;

  

   // push elementi na priorities

   priorities.push( 3.2 );

   priorities.push( 9.8 );

   priorities.push( 5.4 );

 

   cout << "Pop od priorities: ";

  

   while ( !priorities.empty() ) {

      cout << priorities.top() << ' ';  // prikazi go elementot na pocetokot

      priorities.pop();                 // otstrani go

 

   }

 

   cout << endl;

 

   return 0;

 

} // kraj na main

 

Излез од програмата:

Pop od priorities: 9.8 5.4 3.2
