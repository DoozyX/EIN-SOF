4. Standard Template Library

 

 

 

4.1 Вовед

 

Standard Template Library (STL) е дел од стандардната библиотека на C++. Таа содржи множество од компоненти базирани на template-и кои имплементираат голем број на познати податочни структури и алгоритми за обработка на тие структури.

STL се состои од три вида компоненти: контејнери, итератори и алгоритми. Контејнерите се компонени кои можат да складираат објекти од било кој податочен тип. Поделени се во три категории: first-class containers, adapters и near containers. Секој контејнер има соодветни функции членки. Подмножество од овие членки е дефинирано кај сите контејнери во STL.

Итераторите, кои имаат слични својства со покажувачите, се употребуваат за манипулација со елементите на STL контејнерите. Манипулирање на контејнери со итератори заедно со STL алгоритмите дава голема моќ на изразување и го намалува кодот.  Постојат пет категории на итератори.

STL алгоритмите се функции кои извршуваат стандардни операции со податоци, како што е пребарување, сортирање и споредување на елементи. Има приближно 70 алгоритми во STL. Повеќето од нив користат итератори за пристап до контејнерите. Секој алгоритам има свои барања за особините на типот на итераторот кој може да се користи со него. Итераторите го енкапсулираат механизмот за пристап до елементите на контејнерот. Оваа енкапсулација овозможува многу STL алгоритми да бидат применети на низа контејнери без оглед на нивната имплементација.

Овој приод овозможува пишување на општи програми во кои кодот не зависи од контејнерот што се користи. Овој начин на програмирање се нарекува генеричко програмирање.

Имплементирање на сложени динамички структури на податоци, како што се листи, стебла, складови итн., е сложено. Кога се имплементираат големи проекти на кои работат многу програмери, употребата на STL компонентите овозможува code reuse, а со тоа го намалува времето потребно за развој, тестирање и остранување на грешки.  

 

 

4.2 Контејнери

 

STL типовите на контејнери се поделени во три групи: секвенцијални (sequence containers), асоцијативни (associative containers) и адаптери (container adapters).

Секвенцијалните контејнери претставуваат линеарни податочни структури, како што се вектори и поврзани листи. Асоцијативните контејнери се нелинеарни контејнери кои овозможуваат брзо лоцирање на елементите во контејнерот. Тие складираат множества од парови клуч-вредност. Секвенцијалните и асоцијативните контејнери заедно се нарекуваат првокласни контејнери (first class containers).

Складовите и редиците се ограничени верзии на секвенцијални контејнери. Затоа, нивната имплементација во STL е во вид на адаптери кои овозможуваат програма да употребува секвенцијален контејнер.

Постојат четири типови на контејнери кои се сметаат за near containers: полиња (arrays), string, bitset кои сместуваат знаменца и valarray за брзи математички векторски операции. Овие типови се нарекуваат near containers бидијќи имаат можности слични на првокласните контејнери, но не ги поддржуваат сите нивни можности.

 

4.2.1  Секвенцијални контејнери

STL содржи три секвенцијални контејнери: vector, deque и list.

  • vector - Класата vector class овозможува случаен пристап до елементите во контејнерот. Оптимизирана е за брзо вметнување и бришење на елементите на крајот. Сите пристапуваа до елементите се извршуваат за константно време. За вметнување и бришење на елементи на средина е потребно линеарно време.
  • deque - deque често се нарекува и deck. Овозможува случаен пристап до елементите за константно време. Оптимизирана е за брзо вметнување и бришење на елементите на почетокот и на крајот. За вметнување и бришење на елементи на средина е потребно линеарно време.
  • list - Класата list не овозможува случаен пристап до елементите. Листата е имплементирана преку двојно поврзана структура во која секој елемент има покажувач кон претходниот и наредниот елемент. Вметнување и бришење се врши за константно време бидејќи се одвива преку промена на покажувачите, без преместување на самите елементи.

 

Следниот пример ја илустрира употребата на контејнерите. Во него се применуваат итератори и алгоритми, за кои ќе стане збор подетално понатаму во текстот.

  • Креирање на вектор е едноставно. default консрукторот креира празен вектор. Push_back() додава елементи на крај на векторот.
  • Векторот се сортира преку еден повик на генеричкиот алгоритам sort.
  • Векторот се инвертира преку еден повик на генеричкиот алгоритам reverse.
  • sort() и reverse() како аргументи користат итератори. Методата begin() на класата vector враќа итератор на почетокот на векторот; end() враќа итератор една позиција после крајот на векторот.

#include <vector>

#include <iostream>

#include <algorithm>

using namespace std;

 

 

int main( )

{

    // Kreira prazen vektor

    vector <int> v1;

 

    // size_type vo zaglavnata datoteka na vector

  

    int i;

 

    // Dodaj elementi na krajot na v1

    v1.push_back(5);

    v1.push_back(12);

    v1.push_back(1);

    v1.push_back(392);

 

    // Otpecati go vektorot

    for (i = 0; i < v1.size(); i++) {

        cout << v1[i] << " ";

    }

    cout << endl;

 

    // Sortiranje

    sort(v1.begin(),v1.end());

 

    // Otpecati go vektorot

    for (i = 0; i < v1.size(); i++) {

        cout << v1[i] << " ";

    }

    cout << endl;

 

    // Invertiranje

    reverse(v1.begin(),v1.end());

 

    // Otpecati go vektorot

    for (i = 0; i < v1.size(); i++) {

        cout << v1[i] << " ";

    }

    cout << endl;

    return 0;

}

4.2.2  Асоцијативни контејнери

Вториот тип на контејнери се асоцијативни контејнери. Тие овозможуваат конструирање на hash-ови и множества. hash е колекција на парови клуч/вредност. Секоја вредност може да биде вметната преку клуч и пронајдена преку него. Сите асоцијативни контејнери во STL се сортирани, што обезбедува брзо пребарување. Следните асоцијативни контејнери се  дел од STL:

  • set – Содржи единствен клуч за било кој тип. Обезбедува брзо пребарување и тестирање на постоење на клучеви.
  • multiset - multiset исто така содржи клуч за било кој тип, но тие не мора да бидат единствени.
  • map – контејнерот map е hash кој содржи пресликување еден-на-еден. Складира парови клуч/вредност од призволен тип.
  •  multimap – содржи пресликување еден-на-многу. Обезбедува складирање базирано на клучеви кои може да бидат и дуплирани.

Следува едноставен пример кој користи класа set. Повторно, објаснувањето на некои делови од кодот ќе следи во продолжение.

  • Постапката за креирање на множество е едноставна. Во примерот се креира множество од стрингови.
  • Додавањето на елементи е едноставно
  • За пребарување се користи функцијата find. Таа е функција членка на  set.

 

#include <set>

#include <algorithm>

#include <iostream>

#include <string>

 

using namespace std;

 

int main( )

{

 

  // Kreira prazno mnozestvo

    set <int> celi;

 

    // Dodava elementi vo mnozestvoto

    celi.insert(1);

    celi.insert(2);

    celi.insert(3);

    celi.insert(4);

    celi.insert(5);

 

    // find vrakja iterator na pronajden element.

    set <int>::iterator iter = celi.find(7);

 

    if (iter != celi.end()) {

        cout << "Ima" << endl;

    }

    else {

        cout << "Nema." << endl;

    }

 

    return 0;

}

 

Излез од програмата:

 

Nema.

4.3 Итератори

 

Итераторите имаат слични особини со покажувачите и се користат за покажување на елементи кај првокласните контејнери. Тие се имплементираат посебно за секој вид на контејнер. Некои оператори функционираат исто кај сите контејнери (Пр: * и ++)

Првокласните контејнери имаат функции членки begin() и end(). begin() враќа итератор кој покажува на првиот елемент од контејнерот.  end() враќа итератор кој покажува на првиот елемент по последниот елемент на контејнерот (елемент кој всушност не постои).

Ако итератор i покажува на некој елемент, тогаш *i го означува тој елемент, а ++i покажува на следниот елемент. Итераторот кој е резултат на end() може да се користи само за споредба со друг итератор кој се менува, за да се провери дали тој дошол до крај на контејнерот.

 За референца на елементи на контејнери кои може да се менуваат се користи објект од тип iterator. За референца на елементи на контејнери кои не може да се менуваат се користи објект од тип const_iterator.

Итераторите се користат со секвенци, кои можат да бидат во контејнери, или да бидат влезни или излезни секвенци. Следниот пример демонстрира влез од стандарден влез и излез на стандарден излез преку итераторите istream_iterator и ostream_iterator.

 

// Fig. 21.5: fig21_05.cpp

// Demonstracija na vlez i izlez so iteratori.

#include <iostream>

 

using std::cout;

using std::cin;

using std::endl;

 

#include <iterator>  // ostream_iterator i istream_iterator

 

int main()

{

   cout << "Vnesi dva celi broja: ";

 

   // kreiraj istream_iterator za citanje na vrednosti od cin

   std::istream_iterator< int > inputInt( cin );

 

   int number1 = *inputInt;  // citaj int od standard input

   ++inputInt;          // zgolemi go iteratorot

   int number2 = *inputInt;  // citaj od standard input

 

   // Kreiraj ostream_iterator za zapisuvanje na int na cout

   std::ostream_iterator< int > outputInt( cout );

 

   cout << "Sumata e: ";

   *outputInt = number1 + number2;  // prikazi rezultat na cout

   cout << endl;

 

   return 0;

 

} // kraj na main

 

Итераторот inputInt се дереференцира за да се прочита int вредност од cin која се доделува на променливата number1. Операторот за дереференцирање * применет на inputInt ја зема вредноста од потокот асоциран со inputInt. Слично е и за втората вредност.

Се креира ostream_iterator кој може да сместува int вредности на cout. Со:

*outputInt = number1 + number2;

се сместува вредност на cout.

 

Во STL постојат следните категории на итератори:

 

input – се користи за читање на елемент од контејнер. Може да се движи само нанапред. Истиот итератор не може да се употреби за поминување на низа двапати.

output - се користи за запишување на елемент во контејнер. Може да се движи само нанапред. Истиот итератор не може да се употреби за поминување на низа двапати.

 

forward – ги комбинира можностите на input и output итератори.

bidirectional - ги комбинира можностите на forward итератор со можност за движење наназад.

random access - ги комбинира можностите на bidirectional итератор со можност за случаен пристап.

 

Од категоријата на итератори што ја поддржува еден контејнер зависи дали тој може да се користи со одреден STL алгоритам. Контејнерите што поддржуваат случаен пристап можат да се користат со сите алгоритми во STL.

4.4 Алгоритми

 

 

Една од најважните компоненти на STL се алгоритмите кои можат да се користат генерички низ множество од контејнери. Овие алгоритми оперираат врз контејнерите индиректно пеку употреба на итератори. Многу од нив како аргументи примаат пар итератори, првиот покажува кон првиот елемент од низата, а вториот кон последниот. Можно е да се дефинираат и сопствени алгоритми кои оперираат слично на STL алгоритмите.

Честопати резултат на алгоритмите е итератор (На пр. find() враќа локација на елемент).

Контејнери кои поддржуваат помоќни итератори можат да се користат со алгоритми кои употребуваат послаби итератори. Некои алгоритми бараат моќни итератори, на пр.sort() бара random-acess итератор. Бидејќи vector содржи random-acess итератор, следниот код се компајлира и работи коректно:

 

 

 

#include <iostream>

#include <algorithm>

#include <vector>

 

using namespace std;

 

void main() 

{

vector< int > years; 

   years.push_back( 1962 ); 

   years.push_back( 1992 ); 

   years.push_back( 2001 ); 

   years.push_back( 1999 ); 

   sort( years.begin(), years.end() ); 

   vector< int >::iterator i; 

   for ( i = years.begin(); i != years.end(); ++i )   

         cout << *i << "\n"; 

}

 

Излезот е:

 

1962

1992

1999

2001

 

Ако sort() се употреби со послаб итератор, се јавува грешка при компајлирање или поврзување. Следниот пример го илустрира тоа, се јавува грешка при компајлирање бидејќи list поддржува само bidirectional итератори.

 

#include <iostream>

#include <algorithm>

#include <list>

 

using namespace std;

 

void main() 

{ 

   list< int > years; 

   years.push_back( 1962 ); 

   years.push_back( 1992 ); 

   years.push_back( 2001 ); 

   years.push_back( 1999 ); 

   sort( years.begin(), years.end() ); 

   list< int >::iterator i; 

   for ( i = years.begin(); i != years.end(); ++i )   

         cout << *i << "\n"; 

}
